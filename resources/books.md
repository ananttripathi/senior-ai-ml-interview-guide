# Books & Reading List

Organized by priority. Start with Tier 1 if time is limited.

---

## Tier 1: Essential (Read These First)

### ML System Design
**Designing Machine Learning Systems** — Chip Huyen (O'Reilly, 2022)
- The single best book for ML system design interviews
- Covers data engineering, feature engineering, training, deployment, monitoring
- Read chapters 1–7 at minimum; chapter 9 (continual learning) and 10 (infrastructure) if time allows
- Pairs directly with `system-design/ml-system-design-framework.md`

**Machine Learning Interviews Book** — Chip Huyen (free online)
- Specifically written for ML interview prep
- URL: huyenchip.com/ml-interviews-book
- Strong on ML fundamentals Q&A

### Practical ML
**Hands-On Machine Learning with Scikit-Learn, Keras & TensorFlow** — Aurélien Géron (O'Reilly, 3rd ed. 2022)
- Best practical reference for ML implementation
- Read chapters 1–10 for fundamentals; chapters 11–16 for deep learning
- Has good worked examples for interview coding problems

---

## Tier 2: Deep Technical Foundation

### Statistical Theory
**The Elements of Statistical Learning** — Hastie, Tibshirani, Friedman (free PDF)
- Deep mathematical foundations for ML
- Key chapters: 2 (overview), 4 (linear methods), 7 (model assessment), 10 (boosting), 15 (random forests)
- More reference than cover-to-cover read

**Pattern Recognition and Machine Learning** — Bishop
- Bayesian perspective on ML
- Most relevant: chapters 1–4 (probability, linear models) and 8–9 (graphical models, EM)

### Deep Learning
**Deep Learning** — Goodfellow, Bengio, Courville (free online)
- The theoretical bible for deep learning
- Key chapters: 6 (MLPs), 7 (regularization), 8 (optimization), 10 (RNNs), 12 (applications)

### Statistics / Experimentation
**Trustworthy Online Controlled Experiments** — Kohavi, Tang, Xu (Cambridge, 2020)
- The A/B testing book. Written by engineers who ran experiments at Microsoft, Google, Amazon.
- Must-read for Data Scientist roles; important for all
- Key chapters: 1–5 (foundations) and 17–21 (pitfalls and advanced topics)

## Tier 3: Role-Specific

### For ML Engineers
**ML Engineering** — Andriy Burkov (free online)
- Practical ML engineering patterns
- Short and dense — can read in a weekend

**Building Machine Learning Pipelines** — Hapke & Nelson (O'Reilly)
- TFX-focused but concepts apply broadly
- Good for pipeline design questions

### For AI Engineers (LLMs)
**Building LLMs for Production** — various authors (free PDF, BrevDev)
- RAG, fine-tuning, agents, evaluation
- More practical than theoretical

**LLM Engineer's Handbook** — Iusztin & Labonne (Packt, 2024)
- End-to-end LLM pipeline from training to production
- Good for understanding the full stack

### For Data Scientists
**Causal Inference: The Mixtape** — Cunningham (free online)
- Causal inference methods: DiD, IV, RDD, matching
- Increasingly important for senior DS roles

**Mostly Harmless Econometrics** — Angrist & Pischke
- The academic reference for causal inference
- More rigorous than Mixtape

## Must-Read Papers

### Foundational
- **Attention Is All You Need** (Vaswani et al., 2017) — Transformer architecture
- **BERT: Pre-training of Deep Bidirectional Transformers** (Devlin et al., 2018)
- **Language Models are Few-Shot Learners** (GPT-3, Brown et al., 2020)
- **Training language models to follow instructions** (InstructGPT, Ouyang et al., 2022)

### Recommendation Systems
- **Deep Neural Networks for YouTube Recommendations** (Covington et al., 2016)
- **Wide & Deep Learning for Recommender Systems** (Cheng et al., 2016)
- **Neural Collaborative Filtering** (He et al., 2017)
- **Real-time Personalization using Embeddings** (Airbnb, 2018)

### Feature Engineering & Ranking
- **Deep & Cross Network for Ad Click Predictions** (Wang et al., 2017)
- **Feature Hashing for Large Scale Multitask Learning** (Weinberger et al.)

### Production ML
- **Hidden Technical Debt in Machine Learning Systems** (Sculley et al., 2015) — essential read
- **Rules of Machine Learning** — Martin Zinkevich (Google, free)
- **Scaling Machine Learning at Uber** (Michelangelo blog post)
